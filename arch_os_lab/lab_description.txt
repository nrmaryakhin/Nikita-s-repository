Описание работы
1. Создаются папки LOG и BACKUP
2. Проверяется работа скрипта на пустой папке LOG
3. Добавляются текстовые файлы, не превышающие порог архивации и проверяется
работа скрипта в этом случае
4. Добавляются еще текстовые файлы, превышающие порог архивации, запускается
скрипт, устанавливается значение для архивации N первых файлов по дате 
добавления. Эти N фалйлов удаляются из LOG и архивируются в BACKUP
5. Проверка количества файлов в архиве BACKUP, смотрим, что архивировалось
то количество файлов, которое мы указывали 
6. Запускаем скрипт еще раз, опять указываем N. Создается другой архив в 
BACKUP и следующие N файлов удаляются из LOG
7. Еще раз запускаем скрипт - использование папки LOG в норме
8. Проверям папку LOG, мы создавали 8 файлов, в первый раз архивировали 
3 файла, и во второй раз тоже 3. Итого, в папке LOG  осталось 2 файла
*9. Изменен порядок работы с архивацией. Теперь количество файлов для архивации запрашивается, если заполненность папки LOG превышает установленного значения
